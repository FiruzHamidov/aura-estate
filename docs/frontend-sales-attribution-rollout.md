# Frontend Rollout: Sales Attribution (`sale_user_id` + shared deals)

Дата: 2026-05-29

Этот репозиторий содержит backend API и контракты. UI-код отчетных экранов в данном workspace отсутствует, поэтому ниже зафиксирован точный фронтовый rollout-план для внешнего frontend-репозитория.

## Scope

- Метрики продаж больше не считаются только целыми.
- Для совместных сделок backend отдает дробную атрибуцию (`1/N`).
- Поля без переименования: `sold`, `rented`, `closed`, `sales_count`, `deals_count`.
- Семантика изменилась: эти поля могут быть `float`.

## Endpoint Matrix

| Endpoint | Текущая логика на UI (что было рискованно) | Целевая логика на UI | Какие тесты добавить |
|---|---|---|---|
| `GET /api/reports/properties/manager-efficiency` | Часто отображение как int, возможное округление/обрезка | Показывать `sold/rented/closed` как дробные, формат до 2 знаков | Рендер `0.5`, `0.33`; отсутствие `NaN`; корректный total |
| `GET /api/reports/properties/agents-leaderboard` | Сортировка могла быть строковой/целочисленной | Сортировка строго по float, отображение с trim trailing zeros | Сорт: `1.5 > 1.25 > 1.0 > 0.5` |
| `GET /api/reports/agent/earnings` | `closed_count/deals_count` могли рендериться как int | Поддержка дробных `closed_count/deals_count`, корректные подписи/тултипы | 2 участника => `0.5`; агрегаты не теряют дробную часть |
| `GET /api/kpi/daily?date=...&v=2` | KPI sales-блок мог ожидать int | Рендер `metrics.sales.final_value`/`sales_count` как float | `0`, `0.5`, `0.3333` без глитчей |
| `GET /api/kpi/weekly?...&v=2` | Неполная консистентность с reports | Единые форматтеры для weekly sales метрик | Совпадение смысла с manager-efficiency на одинаковом scope |
| `GET /api/kpi/monthly?...&v=2` | Аналогично weekly | Единые форматтеры и сортировки для monthly | Снапшот с дробными значениями, без int-cast |
| Любой CSV/Excel export для указанных экранов | Потеря дробной части при сериализации | Экспортировать raw float, UI-округление не применять к данным | Проверка `0.5`/`0.3333` в файле экспорта |

## UI Contract Rules

- Не приводить sales-метрики к `parseInt`, `Math.floor`, `toFixed(0)`.
- Формат отображения:
  - до 2 знаков после запятой,
  - убирать хвостовые нули (`1.00 -> 1`, `1.50 -> 1.5`).
- В логике сортировки и агрегации использовать raw `number`, не форматированную строку.

## Copy Text For UI

- Tooltip 1: `Совместные сделки делятся между участниками поровну (1/N).`
- Tooltip 2: `Фильтр по агенту в продажах = продавец (атрибуция на backend).`

## Frontend Acceptance Checklist

- Все sales-карточки и таблицы поддерживают дробные значения.
- На одном периоде/фильтре KPI и отчеты не расходятся по смыслу.
- Экспорты сохраняют дробную часть.
- Нет регрессий по фильтрам `agent_id`, `branch_id`, `branch_group_id`.
