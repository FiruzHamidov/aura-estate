# ТЗ: Сторис для пользователей сайта

## 1. Цель
- Добавить функционал сторис, чтобы каждый пользователь сайта мог публиковать сторис.
- Разрешить создавать сторис из:
1. обычного медиа (фото/видео),
2. объявления (property),
3. рилса (reel).
- Поддержать одновременную активацию до 10 рилсов в сторис на одного пользователя.

## 2. Роли и права
- Публиковать сторис может любой авторизованный пользователь со статусом `active`, кроме роли `client`.
- Просматривать сторис могут все пользователи сайта (включая неавторизованных) для публичных профилей.
- Редактировать/удалять сторис может только автор сторис или админ.
- Роль `client` не может создавать, редактировать, удалять и перепубликовывать сторис.
- МОП (`mop`) может управлять сторис пользователей своей группы (`branch_group_id`):
1. просматривать служебный список сторис группы,
2. архивировать/скрывать сторис участников своей группы,
3. перепубликовывать сторис участников своей группы по согласованному бизнес-процессу.
- РОП и директор филиала (`rop`, `branch_director`) могут управлять сторис всех групп в рамках своего филиала (`branch_id`):
1. просматривать сторис по всем группам филиала,
2. архивировать/скрывать/перепубликовывать сторис в рамках филиала.
- Администратор (`admin`) контролирует и управляет всеми сторис без ограничений по филиалу и группе.
- Для ролей `mop`, `rop`, `branch_director` термин "управлять сторис" строго означает только действия:
1. `archive`,
2. `hide`,
3. `republish`.
- Роли `mop`, `rop`, `branch_director` не могут редактировать контент сторис (caption/items/attachments) и не могут удалять сторис.

## 3. Основные сущности

### 3.1 Story
- `id`
- `user_id` (автор)
- `type` (`media`, `property`, `reel`)
- `caption` (nullable, до 500 символов)
- `status` (`draft`, `active`, `archived`, `deleted`)
- `starts_at` (nullable; если пусто, публикация сразу)
- `expires_at` (обязательно для `active`, по умолчанию `created_at + 24h`)
- `views_count` (кэш-счетчик)
- `created_at`, `updated_at`

### 3.2 StoryItem
- Позволяет делать сторис-карусель (несколько слайдов в одной сторис).
- `id`
- `story_id`
- `position`
- `media_type` (`image`, `video`)
- `media_url`
- `thumbnail_url` (nullable)
- `duration_sec` (для отображения прогресса)
- `meta` (json: фильтры, стикеры и т.д., опционально)

### 3.3 StoryAttachment
- Для связи с объявлениями/рилсами.
- `id`
- `story_id`
- `attachable_type` (`Property`, `Reel`)
- `attachable_id`
- `snapshot_json` (снимок заголовка/цены/превью на момент публикации, чтобы не ломать старые сторис)

### 3.4 StoryView
- Лог просмотров.
- `id`
- `story_id`
- `viewer_user_id` (nullable для гостя)
- `guest_token` (nullable)
- `viewed_at`
- Уникальность: `story_id + viewer_user_id` (или `story_id + guest_token`).

## 4. Бизнес-правила

### 4.1 Публикация
- Любой `active` пользователь может создать сторис.
- По умолчанию сторис живет 24 часа после активации.
- Если исходный `Property`/`Reel` скрыт или удален:
1. уже опубликованная сторис остается доступной до `expires_at` через `snapshot_json`,
2. новая публикация из скрытого/удаленного контента запрещена.
- При смене группы/филиала у сотрудника доступ к его сторис (включая старые/архивные) определяется по текущей привязке пользователя:
1. сторис видят и управляют ими руководители филиала, в котором пользователь состоит сейчас,
2. прошлый филиал/группа после перевода пользователя больше не управляют его сторис, если нет прав `admin`.

### 4.2 Сторис из объявления
- Пользователь выбирает свое или доступное ему объявление.
- В сторис сохраняется ссылка на объявление + snapshot (title, price, currency, photo).

### 4.3 Сторис из рилса
- Пользователь может добавлять рилсы в сторис.
- Лимит: **до 10 одновременно активных рилсов в сторис на одного пользователя**.
- Проверка лимита выполняется при активации сторис типа `reel`.
- Время жизни сторис с рилсом: 1 день (24 часа) после активации, далее сторис автоматически скрывается (архивируется).
- Скрытые сторис с рилсами сохраняются в истории сторис и могут быть перепубликованы владельцем (кроме `client`) или уполномоченными ролями в рамках их зоны доступа.

### 4.4 Лимиты
- Максимум активных сторис на пользователя: 30 (настраиваемо).
- Максимум вложений в одной сторис: 10 элементов.
- Максимальный размер:
1. image: до 10 MB,
2. video: до 100 MB, до 60 сек на item.

### 4.5 Модерация
- Базовый режим v1: постмодерация (жалобы + ручное скрытие админом).
- Опционально v2: премодерация для новых пользователей.

## 5. API (черновой контракт)

## 5.1 Публичные
- `GET /api/stories/feed`
  - Возвращает ленту сторис группами по авторам.
- `GET /api/stories/{story}`
  - Детальная сторис.
- `POST /api/stories/{story}/view`
  - Фиксировать просмотр.

## 5.2 Авторизованные
- `POST /api/stories`
  - Создать сторис (`type`, `caption`, `starts_at`, `items[]`).
- `POST /api/stories/from-property/{property}`
  - Быстро создать сторис из объявления.
- `POST /api/stories/from-reel/{reel}`
  - Быстро создать сторис из рилса (с проверкой лимита 10 активных рилсов).
- `PATCH /api/stories/{story}`
  - Обновить подпись/время/статус.
- `DELETE /api/stories/{story}`
  - Мягкое удаление.
- `GET /api/my/stories`
  - Список моих сторис (active/draft/archive).

## 5.3 Админ
- `GET /api/admin/stories`
- `PATCH /api/admin/stories/{story}/status` (`active|archived|deleted`)

## 6. Изменения БД (миграции)
- `create_stories_table`
- `create_story_items_table`
- `create_story_attachments_table`
- `create_story_views_table`
- Индексы:
1. `stories(user_id, status, expires_at)`,
2. `stories(type, status)`,
3. `story_views(story_id, viewer_user_id)`,
4. `story_views(story_id, guest_token)`.

## 7. Логика планировщика
- Cron-команда: `stories:expire`
- Запуск каждые 5 минут.
- Меняет `status=active` на `archived`, если `expires_at <= now()`.
- Для сторис типа `reel` `expires_at` при активации всегда устанавливается как `activated_at + 24h`.

## 8. Безопасность и доступ
- Проверка владельца на update/delete.
- Проверка видимости связанного `Property`/`Reel` при создании.
- Валидация MIME, размера файла, длительности видео.
- Scope-политики доступа:
1. `mop` только в пределах своей `branch_group_id`,
2. `rop` и `branch_director` в пределах своего `branch_id`,
3. `admin` без ограничений.
- Rate limit:
1. создание сторис: `30/час` на пользователя,
2. просмотры: `120/мин` на IP/guest_token.

## 9. Аналитика
- Метрики:
1. количество публикаций сторис в день,
2. досмотры сторис,
3. CTR переходов из сторис в объявление,
4. CTR переходов из сторис в рилс.

## 10. Этапы реализации

### Этап 1 (MVP)
- Таблицы + модели + базовый CRUD сторис.
- Публикация `media` и `property`.
- Публичная лента и просмотры.

### Этап 2
- Публикация `reel` + лимит 10 активных рилсов.
- Архивация по cron.
- Snapshot-данные для устойчивости контента.

### Этап 3
- Жалобы, модерация, аналитика.
- Улучшение UX (прогресс-бар, авто-переходы, рекомендации).

## 11. Критерии приемки
- Любой `active` пользователь, кроме `client`, может создать сторис.
- Можно создать сторис из объявления и из рилса.
- Для рилсов нельзя превысить лимит 10 активных одновременно.
- Сторис с рилсом живет 24 часа и автоматически скрывается.
- Скрытая сторис хранится в истории и доступна для перепубликации по правам доступа.
- Сторис автоматически архивируются после `expires_at`.
- Просмотры фиксируются без дублей для одного viewer.
- Публичная лента корректно показывает только `active` и неистекшие сторис.
